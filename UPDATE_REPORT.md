# AIName 本次更新报告

更新日期：2026-06-21

## 更新概述

本次更新主要完善了管理员端、账号安全、敏感配置管理和项目部署文档，使项目从基础功能原型进一步向可管理、可部署的完整应用靠近。

## 主要更新内容

### 1. 新增管理员端

- 新增独立的管理员登录页面和用户管理页面。
- 普通登录页支持通过隐藏互动进入管理员入口：4 秒内连续点击标题 7 次。
- 管理员身份由后端数据库权限字段校验，隐藏入口本身不作为安全依据。
- 支持搜索和查看普通用户。
- 支持冻结、解冻、重置密码和软删除账号。
- 普通用户不能登录管理员端，管理员也不能通过普通用户入口登录。

### 2. 增强账号安全

- 新增管理员、冻结、删除和 Token 版本等用户状态字段。
- 账号被冻结或删除后，已有登录凭证会立即失效。
- 管理员重置用户密码后，用户旧 Token 会失效，需要重新登录。
- 新增数据库迁移和管理员账号创建脚本。

### 3. 统一敏感配置

- 清除了后端源码中硬编码的 DeepSeek Key、邮箱授权码、JWT 密钥及数据库密码。
- 所有配置统一通过 `settings` 模块读取。
- 新增本地 `.env` 自动加载功能，配置一次后即可直接启动项目。
- `.env` 已加入 `.gitignore`，不会被提交到 Git。
- 保留 `.env.example` 作为不含真实密钥的配置模板。

### 4. 完善项目文档

- 新增 `requirements.txt`，记录后端 Python 依赖。
- 新增 `README.md`，包含环境要求、依赖安装、数据库初始化、管理员创建、前后端启动和 Git 提交说明。
- 清理了可能保留旧敏感常量的 Python 缓存。

## 使用前注意事项

- 在 `.env` 中填写重新生成的 DeepSeek Key、QQ 邮箱授权码、JWT 密钥以及实际数据库连接信息。
- 执行 `alembic upgrade head` 更新 MySQL 用户表结构。
- 执行 `python init_pg_memory.py` 初始化 PostgreSQL 会话记忆表。
- 使用 `python create_admin.py` 创建管理员账号。
- 确认 MySQL、PostgreSQL、Redis 和 Ollama 均已启动后，再运行后端服务。
